# 技术负责人

## 角色定位

技术负责人负责带领开发团队落实架构方案、把控代码质量、协调技术任务和解决跨端技术阻塞，是架构设计与日常研发之间的执行桥梁。

## 核心职责

- 将产品需求拆解为前端、后端、测试和部署任务。
- 组织技术评审、代码评审、接口评审和疑难问题排查。
- 统一编码规范、分支策略、提交规范和测试要求。
- 跟踪技术风险，推动关键缺陷和阻塞问题闭环。
- 指导团队成员理解项目结构、权限模型和工程实践。

## 主要产出物

- 技术任务拆解。
- 代码评审意见和质量改进清单。
- 接口联调计划和风险清单。
- 技术问题复盘。
- 团队研发规范。

## 参与阶段

- 需求评审。
- 技术方案制定。
- 研发执行。
- 联调测试。
- 上线和复盘。

## 与本项目的关注点

- 需要确保前端动态路由和后端菜单权限模型联动正确。
- 需要把控 TypeScript 类型、接口封装、实体关系、异常处理和测试覆盖。
- 需要关注密码加密、JWT、Redis 在线用户、Swagger 文档和部署配置之间的协同。
- 需要推动管理后台基础模块先稳定，再扩展具体业务模块。

## 协作接口

- 与架构师确认技术方向和边界。
- 与项目经理同步进度、风险和阻塞。
- 与产品经理澄清需求细节和验收标准。
- 与各开发角色协调任务、联调和代码评审。
- 与测试确认提测质量和缺陷优先级。
