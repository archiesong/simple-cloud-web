# 产品经理

## 角色定位

产品经理负责定义 simple-cloud-web 的业务目标、用户场景、功能范围和验收标准，保证管理后台真正支撑系统管理、权限管理和运营管理需求。

## 核心职责

- 梳理目标用户、使用场景和核心业务流程。
- 设计用户、角色、菜单、权限、登录、在线用户等后台能力的功能规则。
- 编写需求文档、原型说明、字段说明和验收标准。
- 处理需求优先级、范围变更和版本规划。
- 组织需求评审，确保设计、开发、测试对需求理解一致。

## 主要产出物

- PRD 或需求说明。
- 用户流程图和页面原型说明。
- 权限矩阵和角色操作规则。
- 字段字典、状态字典和异常提示文案。
- 验收标准和边界场景清单。

## 参与阶段

- 需求调研。
- 方案设计。
- 原型评审。
- 开发答疑。
- 验收和迭代优化。

## 与本项目的关注点

- 管理后台核心是 RBAC，产品经理需要明确角色与菜单、按钮权限、数据范围之间的规则。
- 登录、验证码、公钥获取、用户信息、动态菜单等流程需要定义清晰的成功和失败反馈。
- 用户管理、角色管理、菜单管理等页面需要明确列表字段、筛选条件、创建编辑规则和删除约束。
- 中英文国际化文案已经存在，需要关注中文和英文菜单、错误提示、登录文案的一致性。

## 协作接口

- 与 UI 设计工程师确认页面布局、交互细节和文案层级。
- 与前端开发确认页面状态、路由行为、表单校验和空状态。
- 与后端开发确认接口字段、业务规则和错误码。
- 与测试确认验收用例和边界条件。
- 与项目经理确认优先级、范围和发布时间。
