# 安全工程师

## 角色定位

安全工程师负责 simple-cloud-web 的认证授权、数据保护、接口安全、配置安全和上线安全审查，降低后台系统被滥用或攻击的风险。

## 核心职责

- 审查登录、验证码、密码加密、JWT、刷新策略和退出登录流程。
- 评估 RBAC 权限控制、菜单权限、按钮权限和接口权限一致性。
- 检查敏感配置、密钥、证书、数据库账号和 Redis 访问安全。
- 识别接口重放、越权、注入、XSS、CSRF、弱密码和日志泄露风险。
- 制定安全测试、加固建议和上线安全检查清单。

## 主要产出物

- 安全审查报告。
- 权限和认证风险清单。
- 安全测试用例。
- 配置安全和密钥管理建议。
- 上线安全检查清单。

## 参与阶段

- 架构和方案评审。
- 登录权限模块开发前后。
- 联调测试。
- 上线安全审查。
- 安全事件响应和复盘。

## 与本项目的关注点

- 后端包含 JWT 中间件、RSA/AES 工具、密码哈希、验证码、Redis 登录态和公私钥文件，需要严格审查密钥生命周期。
- 前端请求封装包含加密和 Token 处理，需要避免敏感信息在浏览器存储和日志中暴露。
- Swagger、错误信息和日志输出在生产环境中需要控制暴露范围。
- 权限校验不能只依赖前端菜单隐藏，后端接口也必须具备授权保护。

## 协作接口

- 与架构师确认整体安全模型。
- 与后端开发确认认证授权、加密和接口防护。
- 与前端开发确认 Token 存储、请求加密和敏感信息处理。
- 与运维确认密钥管理、网络访问控制和日志策略。
- 与测试确认安全测试和越权测试覆盖。
