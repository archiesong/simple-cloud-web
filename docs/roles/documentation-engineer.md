# 文档工程师

## 角色定位

文档工程师负责 simple-cloud-web 的需求、设计、接口、部署、使用和运维文档体系，确保团队知识可传递、可检索、可维护。

## 核心职责

- 建立项目文档目录、命名规范、模板和维护流程。
- 整理需求说明、架构设计、接口文档、部署文档和用户手册。
- 将代码中的关键约定转化为可读文档，例如路由规则、权限模型、配置项和脚本说明。
- 维护变更记录、版本说明和常见问题。
- 推动文档评审，保证文档与代码实现同步。

## 主要产出物

- 文档目录和模板。
- 开发者指南。
- 部署和运维手册。
- 权限模型、接口契约和配置说明。
- 用户操作手册和 FAQ。

## 参与阶段

- 项目初始化。
- 需求和方案沉淀。
- 开发过程同步。
- 上线前文档补齐。
- 版本迭代和知识库维护。

## 与本项目的关注点

- 当前项目已有 `docs` 目录，适合继续沉淀角色、需求、计划、架构和上线资料。
- 后端 Swagger 可以作为接口文档来源，但仍需要补充业务含义、权限要求和示例。
- 前端和后端 README 目前偏模板化，需要后续补齐项目级启动、配置和部署说明。
- 角色文档可以进一步扩展为团队职责说明、AI 协作提示词或交付检查清单。

## 协作接口

- 与项目经理确认文档计划、交付节点和维护责任人。
- 与架构师确认架构和技术决策表述。
- 与产品经理确认业务规则和用户说明。
- 与开发角色确认脚本、配置、接口和排障内容。
- 与测试和运维确认验收、上线和故障处理文档。
