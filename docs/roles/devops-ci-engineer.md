# DevOps/CI 工程师

## 角色定位

DevOps/CI 工程师负责 simple-cloud-web 的持续集成、自动化构建、自动化测试、制品管理和发布流水线，提升交付稳定性和效率。

## 核心职责

- 设计前端和后端的 CI 流水线，包括安装依赖、类型检查、Lint、测试和构建。
- 管理构建缓存、制品版本、发布包和环境变量注入。
- 自动化部署后端服务、前端静态资源和数据库迁移步骤。
- 集成代码质量、测试覆盖率、安全扫描和发布审批。
- 建立失败回滚、流水线告警和发布审计记录。

## 主要产出物

- CI/CD 流水线配置。
- 构建、测试和发布脚本。
- 制品管理和版本策略。
- 自动化部署说明。
- 流水线失败排障文档。

## 参与阶段

- 工程规范制定。
- 开发环境搭建。
- 持续集成接入。
- 预发布和生产发布。
- 交付效率优化。

## 与本项目的关注点

- 后端提供 `dev`、`test`、`lint`、`build`、`start` 等脚本，前端提供 `dev`、`build`、`type-check`、`test:unit`、`test:e2e`、`lint` 等脚本。
- 前后端 Node.js 版本要求不同，需要在流水线中明确运行环境。
- 前端和后端依赖管理、缓存策略和构建产物应分别处理。
- 测试环境需要 MySQL、Redis、后端服务和前端静态资源之间的稳定编排。

## 协作接口

- 与技术负责人确认质量门禁和分支策略。
- 与前端开发确认构建命令、环境变量和静态资源路径。
- 与后端开发确认服务启动、测试依赖和部署包结构。
- 与测试确认自动化测试执行时机和报告格式。
- 与运维确认发布目标、回滚策略和权限控制。
