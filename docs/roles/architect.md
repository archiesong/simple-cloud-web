# 架构师

## 角色定位

架构师负责定义 simple-cloud-web 的整体技术架构、模块边界和关键技术决策，确保前后端分离管理后台在可维护性、扩展性、安全性和交付效率之间保持平衡。

## 核心职责

- 设计前端 Vue 管理台、后端 MidwayJS 服务、数据库、缓存和部署环境之间的整体关系。
- 制定模块边界，例如认证授权、系统管理、菜单权限、用户角色、基础服务和通用异常处理。
- 评审关键技术方案，包括 RBAC 权限模型、动态路由、接口加密、JWT、Redis 缓存和 TypeORM 实体关系。
- 识别架构风险，推动复杂模块拆分、接口契约稳定和技术债治理。
- 建立编码规范、目录规范、接口规范和跨端协作约定。

## 主要产出物

- 系统架构设计文档。
- 模块边界和依赖关系说明。
- API 契约和数据模型设计建议。
- 技术选型和架构决策记录。
- 高风险改造方案和迁移计划。

## 参与阶段

- 项目立项和技术选型。
- 需求拆解和方案评审。
- 关键模块开发前评审。
- 上线前架构风险复核。
- 后续重构和性能优化。

## 与本项目的关注点

- 后端基于 MidwayJS、Koa、TypeORM、MySQL、Redis、JWT 和 Swagger，需要保证接口层、服务层、实体层职责清晰。
- 前端基于 Vue 3、Vite、TypeScript、Pinia、Vue Router、Axios 和 antdv-next，需要保证动态路由、权限控制和请求封装稳定。
- RBAC 是核心能力，菜单、角色、用户、权限标识和路由生成必须形成一致模型。
- 加密、登录态、在线用户和缓存策略需要兼顾安全性与可排障性。

## 协作接口

- 与产品经理确认权限模型、系统菜单和业务边界。
- 与前端开发确认路由、权限、接口返回结构和状态管理方式。
- 与后端开发确认模块分层、实体关系、事务边界和异常处理。
- 与运维和 DevOps 确认部署拓扑、配置管理、日志和监控。
- 与测试确认核心流程、回归范围和风险用例。
