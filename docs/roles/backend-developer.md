# 后端开发

## 角色定位

后端开发负责 `backend` 服务的接口、业务逻辑、数据模型、缓存、认证授权和基础能力实现，保证服务稳定、安全、可测试。

## 核心职责

- 实现 MidwayJS 控制器、服务、DTO、VO、实体和中间件。
- 维护用户、角色、菜单、部门、认证、在线用户等核心模块。
- 设计 TypeORM 实体关系、查询逻辑、事务边界和分页能力。
- 处理 JWT、Redis、验证码、加密、异常过滤和 Swagger 文档。
- 编写单元测试、接口测试和数据迁移说明。

## 主要产出物

- 后端接口和服务实现。
- DTO、VO、实体、枚举和接口文档。
- 数据库表结构和初始化数据说明。
- 单元测试、接口测试和覆盖率报告。
- 后端配置、部署和排障说明。

## 参与阶段

- 需求和架构评审。
- 数据模型设计。
- 接口开发。
- 前后端联调。
- 测试修复、上线和运维支持。

## 与本项目的关注点

- 后端技术栈为 MidwayJS、Koa、TypeScript、TypeORM、MySQL、Redis、JWT、Swagger 和 Jest。
- RBAC 的核心实体包括用户、角色、菜单、部门等，需要保证关系清晰、查询高效、删除约束明确。
- 登录流程包含验证码、RSA/AES 加密、密码哈希、Token 生成和在线用户缓存。
- 接口需要与前端 `/api` 约定、动态菜单结构和权限标识保持一致。

## 协作接口

- 与架构师确认模块边界、实体关系和安全策略。
- 与产品经理确认业务规则、字段含义和异常文案。
- 与前端开发确认接口契约、数据结构和联调节奏。
- 与数据库工程师确认索引、约束和迁移策略。
- 与测试确认接口测试用例、边界场景和缺陷修复。
