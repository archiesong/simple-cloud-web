# 移动端开发

## 角色定位

移动端开发负责移动端访问场景、H5 适配或未来移动应用的实现，确保 simple-cloud-web 的核心能力可以在移动设备上稳定使用。

## 核心职责

- 评估管理后台在移动端、平板端或嵌入式 WebView 中的使用场景。
- 实现移动端页面、H5 适配、响应式交互或移动应用功能。
- 对接后端认证、用户信息、菜单权限和业务接口。
- 处理移动端缓存、登录态、网络异常、页面性能和兼容性问题。
- 参与移动端安全、打包、发布和监控方案设计。

## 主要产出物

- 移动端需求拆解和技术方案。
- H5 或移动应用页面实现。
- 移动端接口调用和登录态处理说明。
- 兼容性测试记录。
- 移动端发布和回滚说明。

## 参与阶段

- 移动端需求评估。
- 接口设计评审。
- 移动端开发和联调。
- 兼容性测试。
- 移动端发布和问题修复。

## 与本项目的关注点

- 当前项目主工程是 Web 管理台，移动端角色更多面向响应式适配、移动审批、移动查看或后续 App 扩展。
- 需要复用后端 JWT、验证码、用户信息和权限接口，但移动端菜单结构可能需要简化。
- 管理后台的大表格、树形权限和复杂弹窗在小屏设备上需要重新设计交互。
- 若采用 H5 方案，应优先和 `frontend/console` 共享 API 类型、请求封装和权限模型。

## 协作接口

- 与产品经理确认移动端是否需要完整后台能力还是只覆盖关键流程。
- 与 UI 设计工程师确认移动端布局、手势和响应式规则。
- 与前端开发确认可复用组件、接口封装和类型定义。
- 与后端开发确认接口兼容性和移动端错误处理。
- 与测试确认设备矩阵、浏览器版本和兼容性范围。
