# 运维

## 角色定位

运维负责 simple-cloud-web 的运行环境、部署发布、配置管理、监控告警、备份恢复和线上稳定性保障。

## 核心职责

- 准备 Node.js、MySQL、Redis、反向代理和日志环境。
- 管理开发、测试、预发布、生产环境配置和密钥。
- 执行部署发布、服务重启、回滚和故障处理。
- 建立日志采集、监控告警、资源容量和可用性检查。
- 制定数据库备份、恢复演练和安全加固方案。

## 主要产出物

- 环境部署文档。
- 配置项和密钥管理清单。
- 发布、回滚和故障处理手册。
- 监控告警规则。
- 备份恢复和容量规划说明。

## 参与阶段

- 环境规划。
- 测试环境搭建。
- 上线准备。
- 发布执行。
- 线上运行和故障响应。

## 与本项目的关注点

- 后端需要连接 MySQL 和 Redis，并依赖 JWT、加密密钥、Swagger、日志等配置。
- 前端构建产物需要部署到静态资源服务或网关，并正确代理 `/api` 请求。
- 登录和权限模块是核心链路，需要重点监控接口错误率、Token 异常、Redis 状态和数据库连接。
- 配置文件中的密钥、数据库账号和 Redis 信息不得直接暴露在公开环境。

## 协作接口

- 与架构师确认部署拓扑、运行依赖和安全边界。
- 与后端开发确认环境变量、服务启动命令和日志位置。
- 与前端开发确认构建命令、静态资源路径和 API 代理。
- 与 DevOps/CI 工程师确认自动化发布和回滚流程。
- 与测试确认测试环境状态、数据准备和验收窗口。
