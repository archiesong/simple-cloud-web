# UI 设计工程师

## 角色定位

UI 设计工程师负责 simple-cloud-web 管理台的视觉规范、交互体验和界面一致性，确保后台工具高效、清晰、可扫描。

## 核心职责

- 设计管理后台的信息架构、页面布局、表格表单、弹窗和权限配置交互。
- 建立颜色、字体、间距、图标、按钮、状态反馈和表单组件规范。
- 输出页面原型、高保真设计稿和交互说明。
- 关注响应式表现、可访问性、异常状态和空状态。
- 与前端开发共同保证设计在 antdv-next 组件体系下可落地。

## 主要产出物

- 页面设计稿和交互原型。
- 设计规范和组件使用说明。
- 图标、插画、空状态和视觉素材。
- 表单、表格、弹窗、权限树等复杂组件设计说明。
- 设计验收清单。

## 参与阶段

- 产品原型阶段。
- 设计评审阶段。
- 前端开发联动阶段。
- 测试验收和视觉走查阶段。
- 后续体验优化阶段。

## 与本项目的关注点

- 当前前端使用 Vue 3、Vite、antdv-next 和 Pro 组件生态，应优先复用已有组件能力。
- 管理后台应强调效率和信息密度，避免营销化页面和过度装饰。
- 用户、角色、菜单、权限配置页面需要关注批量操作、树形结构、表格扫描和表单校验反馈。
- 登录页、仪表盘、异常页和系统设置需要保持统一视觉风格。

## 协作接口

- 与产品经理确认页面目标、信息优先级和操作路径。
- 与前端开发确认组件约束、响应式断点和状态实现。
- 与测试确认视觉验收点和兼容性范围。
- 与架构师确认是否需要沉淀设计系统或组件规范。
- 与项目经理确认设计交付时间和评审节奏。
