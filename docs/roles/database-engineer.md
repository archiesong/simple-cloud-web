# 数据库工程师

## 角色定位

数据库工程师负责 simple-cloud-web 的数据模型、表结构、索引、迁移、备份和查询性能，保障 MySQL 数据层稳定可靠。

## 核心职责

- 设计和评审用户、角色、菜单、部门、字典、岗位等实体对应的数据结构。
- 优化 TypeORM 查询、分页查询、关联查询和索引策略。
- 管理数据库初始化数据、迁移脚本、备份恢复和数据修复。
- 识别慢查询、锁冲突、数据一致性和级联关系风险。
- 制定数据库命名规范、字段规范和变更流程。

## 主要产出物

- 数据模型设计说明。
- ER 图或实体关系说明。
- 数据库迁移和初始化脚本。
- 索引和查询优化建议。
- 备份恢复和数据变更记录。

## 参与阶段

- 数据模型设计。
- 后端开发评审。
- 接口性能测试。
- 上线前数据准备。
- 线上性能优化和数据修复。

## 与本项目的关注点

- 后端使用 TypeORM 和 MySQL，实体关系包括多对多、一对多和基础实体继承。
- 用户、角色、菜单之间的关联关系会直接影响权限查询和动态路由生成性能。
- 删除菜单、角色、用户时需要明确关联约束和业务保护规则。
- 初始化管理员、基础菜单、默认角色和权限标识需要可重复、可追踪。

## 协作接口

- 与架构师确认数据模型边界和一致性策略。
- 与后端开发确认实体映射、查询逻辑和事务边界。
- 与产品经理确认字段含义、枚举值和数据规则。
- 与运维确认备份、恢复和生产数据变更流程。
- 与测试确认测试数据、边界数据和回归数据准备。
