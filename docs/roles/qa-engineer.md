# 测试

## 角色定位

测试负责 simple-cloud-web 的质量保障、测试设计、缺陷跟踪和发布验收，确保核心后台能力符合需求并能稳定运行。

## 核心职责

- 制定功能测试、接口测试、权限测试、兼容性测试和回归测试策略。
- 编写登录、用户、角色、菜单、权限配置等核心流程测试用例。
- 验证前后端接口契约、异常提示、边界条件和权限隔离。
- 跟踪缺陷修复状态，推动复测和回归。
- 建立自动化测试、冒烟测试和上线验收清单。

## 主要产出物

- 测试计划和测试用例。
- 缺陷报告和复测记录。
- 自动化测试脚本。
- 测试报告和质量风险评估。
- 上线验收清单。

## 参与阶段

- 需求评审。
- 测试设计。
- 开发提测。
- 缺陷修复和回归。
- 上线验收和线上验证。

## 与本项目的关注点

- 前端已有 Vitest 和 Cypress 配置，后端已有 Jest 配置，应优先补齐核心流程自动化测试。
- 权限相关测试要覆盖不同角色、菜单可见性、按钮权限、未登录访问和 Token 失效。
- 登录流程需要覆盖验证码错误、密码错误、公钥获取失败、Token 过期和退出登录。
- 用户、角色、菜单删除需要重点验证关联数据和业务约束。

## 协作接口

- 与产品经理确认验收标准和边界场景。
- 与前端开发确认页面状态、测试选择器和端到端测试路径。
- 与后端开发确认接口测试数据、错误码和测试环境配置。
- 与运维确认测试环境可用性和数据恢复方式。
- 与项目经理同步缺陷风险、阻塞和上线建议。
