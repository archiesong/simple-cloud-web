# 前端开发

## 角色定位

前端开发负责 `frontend/console` 管理台的页面、路由、状态、接口调用和用户交互实现，保证前端体验稳定、可维护、易扩展。

## 核心职责

- 实现 Vue 3 页面、组件、布局、路由守卫和动态路由。
- 维护 Pinia 状态、Axios 请求封装、登录态和权限控制。
- 对接后端接口，处理表单校验、错误提示、加载状态和空状态。
- 编写单元测试、组件测试和必要的端到端测试。
- 优化构建、性能、类型安全和前端工程规范。

## 主要产出物

- Vue 页面和业务组件。
- API 调用模块和 TypeScript 类型定义。
- 路由配置、权限控制和状态管理代码。
- 前端测试用例和测试报告。
- 前端构建和部署说明。

## 参与阶段

- 需求和设计评审。
- 接口契约评审。
- 页面开发。
- 前后端联调。
- 测试修复和上线支持。

## 与本项目的关注点

- 前端技术栈为 Vue 3、Vite、TypeScript、Pinia、Vue Router、Axios、Vitest、Cypress 和 antdv-next。
- 动态路由依赖后端菜单接口，菜单字段、组件映射和权限标识必须保持一致。
- 登录流程涉及验证码、公钥、密码加密、Token 持久化和用户信息获取。
- 用户、角色、菜单和权限配置页面需要关注表格、树、弹窗、表单和批量操作的交互稳定性。

## 协作接口

- 与产品经理确认页面规则、字段含义和验收标准。
- 与 UI 设计工程师确认组件使用、状态反馈和响应式规则。
- 与后端开发确认接口路径、请求参数、返回结构和错误码。
- 与测试确认前端测试范围和回归重点。
- 与运维和 DevOps 确认构建产物、环境变量和发布路径。
