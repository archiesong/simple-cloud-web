# 项目经理

## 角色定位

项目经理负责 simple-cloud-web 的交付节奏、任务协调、风险跟踪和跨角色沟通，确保管理后台项目按范围、质量和时间目标推进。

## 核心职责

- 拆解项目计划，明确里程碑、版本目标、交付范围和负责人。
- 跟踪需求、设计、开发、测试、部署和验收状态。
- 管理变更范围，识别延期、依赖阻塞和质量风险。
- 组织评审会议、每日同步、缺陷复盘和上线检查。
- 推动文档、接口、测试结果和上线材料及时沉淀。

## 主要产出物

- 项目计划和里程碑表。
- 任务拆解和负责人矩阵。
- 风险清单和问题跟踪记录。
- 版本发布计划和上线检查清单。
- 项目周报或阶段总结。

## 参与阶段

- 需求启动。
- 计划制定。
- 迭代执行。
- 测试验收。
- 发布上线和复盘。

## 与本项目的关注点

- 项目包含 `backend` 和 `frontend/console` 两个主要工程，需要协调前后端接口联调节奏。
- 权限、登录、菜单、角色、用户管理属于基础能力，缺陷会影响大部分后续功能。
- 数据库、Redis、环境变量、密钥和部署配置需要在开发、测试、生产环境中提前准备。
- 前端 UI、接口契约和后端实体字段变更需要同步记录，避免联调返工。

## 协作接口

- 与产品经理确认需求优先级和验收标准。
- 与架构师确认技术风险和关键路径。
- 与开发角色确认任务拆分、估时和阻塞。
- 与测试确认提测范围、缺陷等级和回归计划。
- 与运维和 DevOps 确认发布时间、回滚方案和环境状态。
