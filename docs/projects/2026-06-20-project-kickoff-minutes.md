## 项目背景

simple-cloud-web 是前后端分离的管理后台系统，主要包含：

- `backend`：基于 MidwayJS、TypeORM、MySQL、Redis、JWT 的后端 API 服务。
- `frontend/console`：基于 Vue 3、Vite、Pinia、Vue Router、antdv-next 的管理台。

当前项目已具备系统管理、认证安全、前端控制台等基础目录结构，但项目文档仍偏模板化。启动阶段需要重点补齐需求范围、接口契约、数据库初始化、测试策略、安全配置和发布流程。

## 当前基线

- 后端核心目录：`backend/src/modules/system`、`backend/src/modules/security`、`backend/src/common`、`backend/src/config`。
- 前端核心目录：`frontend/console/src/views`、`frontend/console/src/api`、`frontend/console/src/router`、`frontend/console/src/store`、`frontend/console/src/layouts`。
- 后端测试已有少量 Jest 单元测试。
- 前端 Cypress 仍为示例测试，端到端测试需要围绕登录、权限和核心 CRUD 补齐。
- 当前检出目录没有 Git 历史，提交规范和分支策略需要在项目启动阶段确认。

## 核心风险

- RBAC 模型风险：用户、角色、菜单、按钮权限、数据范围和动态路由需要形成一致模型。
- 登录认证风险：验证码、公钥、密码加密、Token、Redis 在线用户、退出登录和 Token 过期需要端到端验证。
- 接口契约风险：前后端菜单字段、权限标识、错误码、分页结构和表单校验需要统一。
- 数据库风险：初始化管理员、基础菜单、默认角色、关联删除和索引策略需要可重复、可追踪。
- 安全风险：密钥、数据库账号、Redis 凭据、Swagger 暴露、日志敏感信息和越权访问需要重点审查。
- 交付风险：前后端 Node.js 版本要求不同，CI/CD、测试环境和部署流程需要提前固化。

## 第一轮待决问题

本次启动会需要先确认第一阶段目标：

- A. 先把现有代码跑通并完成本地开发环境。
- B. 做第一版可演示的后台 MVP。
- C. 面向正式上线，补齐安全、测试、部署和文档。
- D. 先做需求和架构梳理，暂不进入开发。
