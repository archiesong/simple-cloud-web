## 项目背景

simple-cloud-web 是前后端分离的管理后台系统，主要包含：

- `backend`：基于 MidwayJS、TypeORM、MySQL、Redis、JWT 的后端 API 服务。
- `frontend/console`：基于 Vue 3、Vite、Pinia、Vue Router、antdv-next 的管理台。

当前项目已具备系统管理、认证安全、前端控制台等基础目录结构。阶段推进口径已从单纯启动梳理调整为“上线准备优先”：先处理安全、配置、构建和质量门禁问题，再组织 MVP 演示范围。

## 核心风险

- RBAC 模型风险：用户、角色、菜单、按钮权限、数据范围和动态路由需要形成一致模型。
- 登录认证风险：验证码、公钥、密码加密、Token、Redis 在线用户、退出登录和 Token 过期需要端到端验证。
- 接口契约风险：前后端菜单字段、权限标识、错误码、分页结构和表单校验需要统一。
- 数据库风险：初始化管理员、基础菜单、默认角色、关联删除和索引策略需要可重复、可追踪。
- 安全风险：密钥、数据库账号、Redis 凭据、Swagger 暴露、日志敏感信息和越权访问需要重点审查。
- 交付风险：前后端 Node.js 版本要求不同，CI/CD、测试环境和部署流程需要提前固化。

## 第一阶段目标结论

本次阶段总结会确认第一阶段按 C 类目标推进：

- 目标：面向正式上线准备，优先补齐安全、测试、部署和文档。
- 第一优先级：处理 RSA 私钥泄露风险、生产环境配置、JWT 白名单匹配、前端请求路径签名、构建产物和质量门禁。
- 第二优先级：明确 RBAC、登录认证、动态菜单和接口契约。
- 第三优先级：在基础风险收敛后再组织后台 MVP 演示。
